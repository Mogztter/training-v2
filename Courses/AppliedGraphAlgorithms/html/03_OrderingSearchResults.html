Ordering Search Results

Pearson similarity overview and problem statement

In this section we’re going to improve the display of search results. When a user searches for businesses we want to order the results so that we show the most relevant results first. To do that we’ll make use of the Pearson similarity metric.

Let’s have a look at the part of the application that we’re going to build. Switch to the CodeSandbox application, select "Business Search", and search for "Pizza" in the category "Restaurants".

You should see something like this:

Instead of showing results in alphabetical order we want to show the most relevant businesses first. Let’s switch back to Neo4j to learn how we can calculate this relevance. Copy the following command into the query pane of the Neo4j Browser and execute it:

:play applied_graph_algorithms/03_search_results.html

Exercise

Now let’s update the application to use this new information. We want to find users

Instead of selecting the reviews for our businesses based on the opinions of users unrelated to user-name, we’ll show reviews written by similar users.

  1. Be sure you’ve completed the Neo4j Browser guide section "Search Results"
  2. In Codesandbox, edit the Cypher query in src/exercises/exercise2.js to return businesses

This query should be be in the context of the currently "logged in" user in the application. Since our simple application doesn’t have any authentication mechanism we fake this by choosing a user in the dropdown in the "User Profile" section of the application. The id of the currently logged in user will be available as a Cypher parameter userId in the query.

So for example, to find the currently logged in user:

MATCH (me:User {id: $userId})
